# SA-project
## 项目背景
 
课设要求实现一个小型智能信息处理系统，涵盖三类算法的模拟实现：
 
- 数据挖掘分类器（有监督学习）
- 基于粗糙集理论的数据分析（属性约简）
- 无监督聚类算法
 
选用微博情感数据集 [`weibo_senti_100k`]（10万条带正/负标签的微博评论）作为实验数据。

 
## 实现的功能
 
### 1. 情感分类器
 
用 jieba 分词 + TF-IDF 特征提取 + 朴素贝叶斯分类器，训练一个能判断评论正负向的模型。
 
- 训练脚本：`train_model.py`
- 预测脚本：`predict.py`
 
 
### 2. 粗糙集关键词分析
 
用粗糙集理论的依赖度公式评估每个词对情感分类的影响力，并通过前向贪婪算法做属性约简，找出"最小有效关键词集合"。
 
- 核心脚本：`roughset_analysis.py`
- 输出：依赖度排名前20关键词柱状图
  
**存在的问题：**
  停用词表不够全时，"啊""哈哈"这类词也可能排名靠前，影响结果可读性。
 
 
### 3. KMeans 文本聚类
 
不使用标签，让模型自动将评论分成 k 堆，再用 t-SNE 降维到二维画散点图。
 
- 脚本：`clustering.py`
- 输出：t-SNE 散点图，不同颜色代表不同聚类
 
**存在的问题：**
  - 聚类数 k=3 是手动指定的，没有数据依据
  - t-SNE 降维损失信息大，图上的点簇不一定对应语义上的真实分组
  - Cluster 0/1/2 没有实际含义，只表示系统认为这几句话比较像
    
这是课设记录，不是生产可用的系统，代码以实现算法流程为主，工程上比较粗糙
